# Three-Stage Pipeline

The three-stage pipeline is the structured production workflow for turning each [[entities/caseware-working-papers]] template into a complete training package.

## Stage 1 — How-To Document (V1)

The base document, created from the original CaseView working paper PDF.

**Covers:**
- Purpose of the working paper
- Field-by-field completion guidance
- Navigation tips within CaseWare
- Practical notes and tips

**Output:** Professionally formatted `.docx` file using the standardised [[concepts/ooxml-document-structure]].

## Stage 2 — Enhanced How-To Document (V2)

The V1 is enhanced with a **"Key Questions — Additional Guidance"** table.

**Process:**
- Cross-reference the working paper against the relevant chapter of the [[entities/probe-audit-manual]]
- Identify the 3–5 most critical questions in the paper
- Build a table with three columns:
  1. The question text
  2. Why it matters (linked to [[concepts/isa-standards]])
  3. What to document

**Output:** Enhanced `.docx` that adds real audit intelligence to the base how-to.

## Stage 3 — Video Script

The V2 document is converted into a narrated presenter script.

**Includes:**
- Production cues for screen recording
- Section transitions
- Video Details metadata table
- Ready-to-record format for CaseWare walkthroughs

**Output:** Script document ready for recording.
