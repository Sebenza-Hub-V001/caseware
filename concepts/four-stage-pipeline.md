# Four-Stage Pipeline

The four-stage pipeline is the structured production workflow for turning each [[entities/caseware-working-papers]] template into a complete training package. Each working paper progresses through all four stages sequentially.

## Stage 1 — How-To Document

The base training document, created from the original CaseView working paper PDF.

**Covers:**
- Purpose of the working paper
- Field-by-field completion guidance
- Navigation tips within CaseWare
- Practical notes and tips
- "Key Questions — Additional Guidance" table cross-referenced against the [[entities/probe-audit-manual]] with [[concepts/isa-standards]] linkage

**Output:** Professionally formatted `.docx` file using the standardised [[concepts/ooxml-document-structure]]. Currently at document version V5.

## Stage 2 — Visual Script

The How-To document is converted into a narrated presenter script for screen-based walkthroughs.

**Includes:**
- Production cues for screen recording
- Section transitions
- Video Details metadata table
- Ready-to-record format for CaseWare walkthroughs

**Output:** Script document ready for recording.

## Stage 3 — Visual Recording

The raw screen recording of the working paper walkthrough in CaseWare, narrated from the visual script.

**Output:** Raw video recording.

## Stage 4 — Professional Video

The final edited and produced training video, ready for distribution.

**Output:** Polished, professional training video.
