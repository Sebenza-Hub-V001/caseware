# OOXML Document Structure

The standardised document formatting used across all How-To documents produced by the [[concepts/three-stage-pipeline]]. All V1 and V2 documents are output as `.docx` files conforming to this structure.

## Formatting Standards

| Element | Specification |
|---------|--------------|
| Heading font | Montserrat |
| Colour palette | Custom branded palette (details TBD) |
| Section numbering | Numbered sections |
| Header/Footer | Branded header and footer |
| File format | `.docx` (OOXML) |

## Consistency

- Every deliverable uses the same fonts, colour scheme, and section structure.
- The formatting is encoded in production skills (`probe-howto`, `probe-enhance`, `video-scripts`) to ensure consistency across all documents.
- This structure applies to both V1 (How-To) and V2 (Enhanced) documents.
