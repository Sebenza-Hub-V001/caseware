# Understanding the Information Technology Environment (IT)

## Purpose

This 4-page working paper documents the auditor's understanding of the entity's IT environment as it relates to financial reporting. It covers IT applications, infrastructure, processes, and an evaluation of the complexity of the IT environment. The conclusion determines whether specialised IT skills are required for the audit.

## Key Fields & Sections

### Risk Assessment Procedures Header
- Type of procedures performed (Inquiries, Observations, Inspections)
- Source(s) from which information was obtained
- Risk indicator column

### Use of Information Technology

#### IT Applications (Q2-3)
- **Primary accounting software** — Software used to initiate, record, process and report transactions for financial statement preparation
- **Other IT applications** — Additional applications used for transaction processing, including:
  - IT applications used in business operations providing data to accounting software
  - Web-facing IT applications
  - IT applications interacting with vendors or external parties via the internet

#### IT Infrastructure (Q4-8)
- **Network infrastructure** — Networks used to transmit/share data, resources and services; operating systems managing hardware/application communications
- **Data storage** — Format and location of digitally stored data:
  - Local, remote, and cloud storage
  - Databases where IT application data is stored
  - Data warehouses (central repositories of integrated data)
  - Report-writers for extracting data from various sources
- **User access to data** — Nature of access:
  - IT personnel with database administration privileges
  - Access through IT applications/data warehouses
  - Direct access without using IT applications
- **Emerging technologies** — Nature and extent of blockchain, robotics, artificial intelligence usage
- **Interfaces** — Nature and extent of data input through interfaces (not manual):
  - Customer or vendor input through application interfaces
  - File uploads

#### IT Processes (Q9-10)
- **IT personnel** — Number and skill level of personnel managing access, security and changes
- **Service providers** — Use of internal or external service providers:
  - Third-party hosting or IT outsourcing
  - Shared service centres for centralised IT management in a group

### Evaluation of Complexity of IT Environment (Q11-21)
Yes/No questions with description and risk fields:

- In-house developed accounting software
- IT applications beyond primary accounting software used for transaction processing
- Mainframe, web-facing elements, external interfaces, cloud services in IT infrastructure
- Direct data access by users without accounting software
- Data stored in databases or data warehouses
- Large volume or complex data processing
- Highly automated transaction processing in financial reporting
- Paperless transaction processing
- Reliance on system-generated reports
- Data captured by external parties or automated procedures (customer/vendor input, file loads, interfaces)
- Emerging technologies affecting financial reporting (blockchain, robotics, AI)

### Conclusion
- Assessment of complexity of the IT environment (simple/moderate/complex)
- Whether specialised IT skills are required, considering:
  - Sufficient understanding of IT applications and environment
  - Determining risks related to use of IT
  - Identifying general IT controls

## Completion Guidance

- Complete during the planning phase, informed by discussions in [[2-working-papers/2-pre-engagement-planning/10-31-discussions-with-tcwg/10-31-discussions-with-tcwg Wiki Page]] (IT Environment section)
- The IT environment understanding drives the identification of general IT controls (GITCs) which are assessed separately
- If the conclusion is that the IT environment is complex, specialised IT skills may be required — this should be factored into the time budget at [[2-working-papers/2-pre-engagement-planning/10-40-time-budget/10-40-time-budget Wiki Page]]
- Responses about data access (Q6, Q14) and automated processing (Q17-20) directly impact the assessment of risks arising from IT
- For entities using service organisations (Q10), consider whether SOC 1/SOC 2 reports are available
- The assessment of complexity feeds into the overall risk assessment at [[2-working-papers/3-planning-risk-assessment/11-10-risk-analysis-summary/11-10-risk-analysis-summary Wiki Page]]
